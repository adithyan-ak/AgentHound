### Issue 1 — release collector version was overwritten by stale per-command constants (fixed and real-harness verified)
- **Symptom:** artifacts from different verbs claimed unrelated development versions (`0.1.0`, `0.2.0-dev`, `0.3.0-dev`, and similar) even when built as one release binary. A consumer could not reliably attribute evidence to the binary that produced it.
- **Root cause:** `common.NewIngestData` correctly injected the process version, but the combined scan, network scan, discover, loot, extract, and campaign envelope builders subsequently overwrote `meta.collector_version` with command-local constants (or redundantly reassigned it).
- **Patch:** remove every verb-level override and preserve the single version installed by the binary entry point through `common.SetCollectorVersion`; there is now one provenance source rather than six drifting copies.
- **Regression:** `TestCollectionEnvelopesPreserveInjectedCollectorVersion` injects `1.0.0-test` and exercises combined scan, network scan, discover, loot, extract, and campaign envelope construction, requiring the exact injected value in each artifact.
- **Verification:** focused tests pass, and the clean 21-scenario real-infrastructure harness's version smoke check passed. The final release tag will still be responsible for injecting the actual GoReleaser version.

### Issue 2 — Open WebUI public identity probes fabricated anonymous privileged access (fixed and composition verified)
- **Symptom:** a real Open WebUI with authentication enabled exposes `/api/version` and `/api/config` publicly, as designed. The fingerprint rule interpreted those responses as `auth_method=none` and `is_anonymous_loot=true`, falsely claiming unauthenticated privileged access.
- **Root cause:** the rule conflated service identification with authorization. Neither public endpoint exercises chat, admin, model-management, or other protected functionality.
- **Patch:** keep the two public probes solely as identity/shape evidence. The graph contribution omits auth fields entirely, while the action summary reports `unknown`; only a looter that observes `/api/config` auth state and protected surfaces authors affirmative auth evidence.
- **Regression:** a protected `httptest.Server` exposes identity/config but returns 401 for `/ollama/config`; fingerprinting must match the service, must not call the protected endpoint, and must emit no `auth_method`, `auth_assurance`, `auth_evidence`, or `is_anonymous_loot`. Same-canonical-endpoint fingerprint+loot regressions separately cover both auth-enabled and auth-disabled composition (Issue 15).
- **Verification:** package race tests pass; the real authenticated Open WebUI harness lane and rebuilt graph/browser representation pass without an anonymous-access fabrication.

### Issue 3 — MCP task capability promised by the graph was silently discarded (fixed across transports)
- **Symptom:** a server can advertise the standard `capabilities.tasks` object in its raw initialize result, but MCP Go SDK v1.6.1 does not expose that field in its typed `InitializeResult`. AgentHound therefore published `has_tasks_capability=false` despite receiving `tasks`, contradicting both wire evidence and the graph contract.
- **Hard constraint / loop break:** replacing the SDK's streamable connection wrapper would break private negotiated protocol/session behavior. The patch instead observes the raw initialize response at the transport boundary while leaving the SDK connection type intact.
- **Patch:** instrument streamable HTTP by cloning its HTTP client/round-tripper and instrument SSE/stdio with a connection wrapper. Correlate only the initialize JSON-RPC ID, cap buffering, parse JSON or complete SSE events, and distinguish an absent key (`false`) from a present object (`true`) and malformed/null/unknown shapes (omit the claim). The observed `tasks` capability is merged back into the server node.
- **Regression:** raw-task tests cover streamable HTTP with negotiated protocol version, session ID, initialized notification, standalone SSE attempt, tool listing, and session delete; legacy SSE; stdio; present/absent/null/non-object semantics; and unchanged headers/lifecycle.
- **Verification:** focused race and vet checks pass, and the clean real MCP harness lanes pass. This is a compatibility observer for the pinned SDK rather than a forked protocol implementation.

### Issue 4 — omitted MCP resource size was fabricated as zero bytes (fixed)
- **Symptom:** resources whose initialize/list response omitted `size` appeared in the graph and UI as `size=0`, an affirmative measurement that the collector never observed.
- **Root cause:** the pinned SDK maps both an absent JSON field and explicit numeric zero to Go `int64(0)`, destroying presence information before AgentHound builds the graph.
- **Patch:** omit `size` when the SDK value is zero; retain nonzero sizes. This favors unknown over fabrication because the current SDK cannot prove whether a zero was explicit.
- **Regression:** one real in-process MCP server exposes an omitted-size resource and a 42-byte resource; the first must have no `size` property and the second must retain `int64(42)`.
- **Verification:** focused race/vet and real MCP collection pass. The explicit-zero ambiguity is documented here as an SDK limitation, not represented as observed data.

### Issue 12 — combined shared-owner fingerprints were hashes of the union, not each owner contribution (fixed and live fallback/APOC verified)
- **Symptom discovered by independent diff review:** the temporary Issue 6 patch computes fingerprints only after `coalesceObservationGraph` has unioned same-ID/triple facts from Config and MCP. It writes the same union digest under every owner. A later exact targeted refresh hashes only that owner's original subset, does not match the stored union digest, and is falsely property-incomplete.
- **Why existing tests missed it:** the integration regression seeded the two owners in separate `WriteObservationNodes` calls. Production `agenthound scan` combines them before the server writer; the test therefore bypassed the exact composition path it intended to prove.
- **Real evidence:** current Config artifacts already contain shared MCPServer/Host rows with 8–9 config ownership domains and unioned properties. The defect is inherent to the pre-write coalescing order, not hypothetical timing.
- **Required patch constraints:** preserve each domain's pre-union fact contribution through Config/MCP/A2A aggregation, compute each semantic digest before the writer unions the public row, keep authoritative and `reference_only` rows separate, remain input-order independent, and fail closed on genuinely conflicting overlapping values. Public graph/write-row totals must remain unique.
- **Regression plan:** one combined two-owner write followed by independent exact refresh of each owner must remain complete; a changed owner must remain incomplete in both input orders; fallback and APOC paths must receive the same per-owner fingerprint set.
- **Patch-review loop break:** the first redesign retained an owner's old stable fingerprint after an incompatible changed refresh. Because the writer also merged dirty candidate properties, a later retry of the old semantic fact could re-certify the union while leaving a newly added dirty property behind. The corrected design removes the incoming owner's stable fingerprint on incompatibility, preserves the last coherent public properties/labels rather than merging an unpublished candidate, and requires either a joint all-owner replacement or ownership retirement before completeness can recover. Exact retries of either unchanged owner still rotate normally.
- **Conflict preservation:** collector aggregation may discard only byte-equivalent duplicate contributions. Non-identical fragments from the same owner domain must reach writer preparation so non-overlapping properties union deterministically and overlapping disagreements fail before any Neo4j execution; last-write-wins coalescing is not acceptable evidence handling.
- **Strengthened recovery regression:** combined A+B complete → independently refresh A and B → changed A (including a new property/label) remains incomplete without mutating the coherent public fact → retry old A remains incomplete → combined A+B replaces/restores the union and fingerprints → retire B leaves A complete. Run for nodes and edges, with the same semantic row parameters under fallback and APOC.
- **Second patch-review catch:** the initial invalidation `CASE` ran for every complete existing-owner refresh, including a fingerprint-compatible one, so the first exact refresh marked the fact complete while deleting its own stable fingerprint. `compatible_existing_owner` now preserves fingerprints before the incompatible-complete invalidation branch in node, fallback-edge, and APOC-edge Cypher. The integration repeats Config exact refresh twice and requires two fingerprints after each, preventing a one-refresh-only false pass.
- **Final implementation:** Config, MCP, and A2A retain distinct owner/semantics contributions and discard only canonically identical duplicates; the pipeline no longer unions them. Writer preparation fingerprints every owner before deterministic compatible union, keeps authoritative/reference-only rows separate, rejects conflicting values/labels/kinds/edge semantics before execution, and writes one logical database row. Volatile observation timestamps are excluded from semantic identity and the latest generated UTC value is retained.
- **Verification:** focused race passed for graph, ingest, Config, MCP, A2A, and SDK ingest; focused vet, `gofmt -l`, and `git diff --check` passed. Live Neo4j 4.4.48 fallback regression logged `APOC not available, using fallback writer`.
